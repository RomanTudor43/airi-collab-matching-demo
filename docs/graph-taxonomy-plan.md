# Graph Taxonomy Plan (CMS-first, runtime intersections)

Date: 2026-04-28

## TL;DR
- sourceKind is the only protection: sourceKind=manual means no pipeline overwrite; all other papers are regenerated each run.
- graphMacroPrimary decides placement; graphMacroTags are cross-discipline highlights only.
- graphMesoTags decide meso placement; pipeline assigns for non-manual papers.
- No fallback to community placement. Pipeline must populate macro/meso for all non-manual papers.
- Intersections are based on cross-links, not tags.
- Meso links are always visible and weighted by strength; hover emphasizes linked nodes and edges.

## Decisions locked
- Intersection view is based on cross-links (GraphLink edges that connect papers in different macros).
- No GraphIntersection content type.
- sourceKind is the master lock: sourceKind=manual means no pipeline overwrite of content or graph relations.
- When sourceKind is not manual, pipeline overwrites macro and meso relations on every rebuild.
- graphMacroPrimary is authoritative for placement. graphMacroTags are optional highlights only.
- No macro/meso colors stored in Strapi; frontend randomizes styling.
- Remove fallback-to-community placement; do not use community as a macro stand-in.

## Current state (as of 2026-04-28)
- Links are generated by full-pair cosine similarity over embeddings; each link is stored once per pair.
- Duplicate pairs are detected and excluded from upload.
- Graph links are replaced on every global rebuild.
- isCrossCluster is computed by algorithmic community, not manual macro.
- Frontend inter-macro bridges are derived from links + graphMacroPrimary.
- Meso view lines are link-based and weight-scaled.
- graphMacroPrimary and graphMesoTags are used for placement.
- Pipeline now writes graphMacroPrimary/graphMacroTags on global rebuild for non-manual papers.

### Implications
- Paper-level links do not depend on macro or meso tags; they depend only on embeddings.
- isCrossCluster will be inaccurate once macros are CMS-driven unless recomputed.

## Target behavior (authoritative)

### Macro assignment (automated unless manual)
Goal: editors do not set graphMacroPrimary manually for thousands of papers.

Rules:
1. For sourceKind=manual, do not overwrite macro relations.
2. For all other papers, pipeline recomputes graphMacroPrimary and graphMacroTags on every global rebuild.
3. graphMacroPrimary decides placement; graphMacroTags do not influence placement.
4. No fallback to community placement in the UI.

Suggested scoring approach:
- Build a macro embedding from GraphMacro fields (name + keywords + description).
- For each paper embedding, compute similarity to each macro embedding.
- Assign the highest score as graphMacroPrimary.
- Assign graphMacroTags for any macro above a configurable threshold, or top-N if needed.

Rationale:
- When macro keywords change, re-running the pipeline realigns papers immediately.

### Meso assignment (automated unless manual)
Goal: meso placement is regenerated with the same policy as macro (sourceKind gate).

Rules:
1. For sourceKind=manual, do not overwrite graphMesoTags.
2. For all other papers, pipeline recomputes graphMesoTags on every global rebuild.
3. graphMesoTags determine meso placement in the UI.

Suggested approach:
- Continue using topic superclusters as the meso basis (already computed).
- Map each topic supercluster to a GraphMeso entry (create or update as needed).
- Use a stable slug or key such as sc-<id> to make updates deterministic.
- Assign GraphMeso.macro by majority of graphMacroPrimary among its member papers.
- Preserve manual fields on GraphMeso (description, sortOrder, isActive) by only updating name/slug/macro/keywords.

Note:
- GraphMeso entries are treated as pipeline-managed unless a manual lock field is added later.

## Intersection view: cross-link based

### Runtime data recipe
1. Resolve macro membership for each paper using graphMacroPrimary only.
2. For every GraphLink edge, map source macro and target macro.
3. If source macro != target macro, count as a cross-macro link.
4. Build macro-bridge edges from these counts.
5. Clicking a macro-bridge opens the two-sided paper view showing only cross-linked papers.

### Display rule
- Only show an intersection view if at least one cross-link exists between the two macros.

### isCrossCluster
- Option A: recompute isCrossCluster based on macroPrimary during rebuild.
- Option B: treat it as legacy and compute cross-macro at runtime only.

## Meso-level links (visible, weighted)

Current: constellation lines are proximity-only and not link-aware.

Target: show link-based meso edges with strength-based styling.

Suggested edge weighting:
- Weight = count of cross-links between papers in meso A and meso B.
- Alternative: sum of link scores or average score.
- Normalize weight to [0, 1] and scale opacity/width.

Hover behavior:
- Hovering a meso node highlights its strongest linked neighbors and their connecting edges.
- Hovering a meso edge highlights both nodes and increases edge opacity.
